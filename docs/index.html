Each account has rate limits. You hit the ceiling, you wait. Meanwhile your teammate's account is doing nothing. HotBunk pools idle capacity so someone is always using it.

// how it works

The submarine metaphor. On submarines, crews share the same bunks across shifts. There aren't enough beds for everyone at once, so when one sailor is on watch, another sleeps in their bunk. The bunk is always "hot" - always in use. HotBunk does this with Claude Code rate limits.

Account States

Interactive Owner is in an active session No automation
Idle No active session detected Within policy
Sleeping Inside declared sleep window Full access
Throttled Hit a rate limit, recovering Backing off
Cooldown Recently throttled, deprioritized Last resort

Interactive sessions have absolute priority. Automated workloads are scavengers. When an owner starts a session, automated work yields immediately.

// architecture

+-----------------------+ | always-on machine | | - orchestrator | | - SQLite state | | - dashboard :3000 | +-----------+-----------+ | mesh network +----------------+----------------+ | | | +-----+-----+ +------+------+ +------+------+ | machine-1 | | machine-2 | | machine-3 | | agent | | agent | | agent | +-----+------+ +-------------+ +-------------+ | interactive sessions here
OrchestratorPython daemon on your always-on machine. Tracks account states, assigns jobs, exposes a REST API and dashboard.
Machine AgentLightweight process on each machine. Reports active sessions, detects which account is in use, receives job assignments.
CLIThe hotbunk command. Submit jobs, check status, switch accounts, edit policies.
DashboardWeb UI showing pool status, active jobs, usage history. FastAPI + HTMX.

// consent model

Trust is the foundation. Every account owner stays in full control.

  • Opt-in only. Nothing runs on your account until you explicitly allow it.
  • Policy-controlled. You choose which job types are allowed, what hours, how many concurrent.
  • Instant revoke. Set allow_automated: false and it stops. Immediately.
  • Full audit trail. Every job assignment is logged. See exactly what ran on your account and when.
  • Interactive priority is absolute. If you start a session, automated work yields. Always. No exceptions.

No credentials are shared between users. Each account's OAuth tokens stay in their own isolated directory with 600 permissions.

// quick start

Install
$ pip install hotbunk
Register your first account
$ claude auth login --email work@example.com $ hotbunk register work --email work@example.com
Register your second account
$ claude auth login --email personal@example.com $ hotbunk register personal --email personal@example.com
See your pool
$ hotbunk status
Submit your first job
$ hotbunk submit militia -c "claude -p 'run the nightly audit'"
